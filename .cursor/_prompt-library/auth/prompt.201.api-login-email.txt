아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TS 파일경로: host/src/pages/api/auth/login.ts

==============================================

핵심요구사항) 이메일/비밀번호 기반 로그인 API를 구현할 것.
            1. 엔드포인트 구조
                - 메서드: POST 단일 엔드포인트
                - 역할: 이메일/비밀번호로 로그인 처리
            2. 요청 데이터
                - email: 사용자 이메일 (필수)
                - password: 사용자 비밀번호 (필수)
                - returnTo: 로그인 성공 후 이동할 경로 
            3. 인증 처리
                - 메서드: signInWithPassword 사용
                - 성공조건: access/refresh token 확보
                - 실패조건: 인증 실패 시 에러 응답 반환
            4. 브릿지 응답
                - 토큰반영: 클라이언트 저장소 업데이트
                - 이동: returnTo 경로로 최종 이동 (없으면 기본 경로)
            5. 에러 처리
                - 유효성 검증: 이메일/비밀번호 필수 체크
                - 인증 실패: 적절한 에러 메시지와 상태 코드 반환
추가-요구사항)
            - returnTo 정책: 상대 경로만 허용
            - 인증옵션: 서버 클라이언트는 세션 자동 저장/자동 갱신 비활성화
            - 클라이언트 생성: _prompt-library/auth/prompt.101.initialize.txt 구현 유틸 import 사용
            - 금지 사항: 각 파일에서 createClient 직접 생성
