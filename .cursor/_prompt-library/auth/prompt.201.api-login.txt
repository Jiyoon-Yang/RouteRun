==============================================

조건-파일경로) 구현될 TS 파일경로: host/src/pages/api/auth/Oath-login.ts

==============================================

핵심요구사항) OAuth 로그인 시작/콜백 API를 구현할 것.
            1. 엔드포인트 구조
                - 메서드: GET + POST 단일 엔드포인트
                - POST 역할: 공급자 로그인 시작 URL 발급
                - GET 역할: OAuth callback(code) 처리
            2. 공급자 분기
                - 기준값: action
                - 기본값: action 미지정 시 기본 공급자 로그인
                - 지원범위: 구글 + 카카오
            3. 콜백 처리
                - 성공조건: code 교환 성공 후 access/refresh token 확보
                - 실패조건: 토큰 누락 시 로그인 실패 경로 처리
            4. 브릿지 응답
                - 토큰반영: 클라이언트 저장소 업데이트
                - 이동: returnTo 경로로 최종 이동
            5. 보조 처리
                - 경로: hash 기반 토큰 전달 케이스 지원
                - 추출값: URL hash의 access_token, refresh_token
                - 저장조건: 토큰 2종이 모두 있을 때만 클라이언트 저장소 반영
                - 실패동작: 토큰 누락 시 로그인 에러 경로로 이동

추가-요구사항)
            - returnTo 정책: 상대 경로만 허용
            - 인증옵션: 서버 클라이언트는 세션 자동 저장/자동 갱신 비활성화
            - 클라이언트 생성: _prompt-library/auth/prompt.101.initialize.txt 구현 유틸 import 사용
            - 금지 사항: 각 파일에서 createClient 직접 생성
