아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/commons/providers/auth/auth.provider.tsx

==============================================

핵심요구사항) 인증 상태 Provider를 구현할 것.
            1. 로그인 처리
                - 입력값: 로그인 성공 후 전달받은 토큰
                - 반영시점: 저장 즉시 인증 상태 업데이트
            2. 로그아웃 처리
                - 정리대상: 토큰 저장소 (로컬 스토리지)
                - 상태변경: 인증 상태 즉시 초기화
            3. 토큰 조회
                - 용도: 내부 API 요청
                - 제공값: 현재 access token
            4. 초기 복원
                - 기준: 앱 시작 시 저장된 토큰
                - 결과: 인증 상태 복원
            5. 제외 범위
                - 리프레시 토큰 기반 자동 재발급 흐름 미포함

추가-요구사항) 설계/의존성 규칙
            - 저장소 키: accesstoken, refreshtoken 사용
            - 로그아웃 API 호출: 옵션 분리 가능 구조
            - Supabase 호출: Provider에서 직접 호출 금지, auth API + 토큰 저장소 경유
            - 금지 사항: 각 파일에서 createClient 직접 생성
