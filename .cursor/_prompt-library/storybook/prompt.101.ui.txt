핵심요구사항)
1) 축 설계(공통 기준)
- 축 우선순위: `variant(최상위) > state/type(하위) > size/theme`
- 필수 축:
  - `variant`: 'primary' | 'secondary' | 'tertiary'
  - `size`: 'small' | 'medium' | 'large'
  - `theme`: 'light' | 'dark'
  - `state`: 'default' | 'hover' | 'active' | 'disabled' (피그마 기준)
- 선택 축(피그마에 있을 때만): `type/intent/shape/fullWidth/iconOnly/nativeType` 등

2) 컴포넌트 API 원칙
- 피그마에 존재하는 항목만 props로 노출.
- 기본값 명시(예: `variant='primary'`, `size='medium'`, `theme='light'`, `state='default'`).
- 타입은 유니온 타입 우선, `any` 사용 금지.
- 접근성 속성(`disabled`, `aria-*`)은 역할에 맞는 최소 범위로 적용.
- 에러·유효성 등 시각이 있으면 루트(또는 해당 역할 요소)에 `aria-invalid` 등 **역할에 맞는 ARIA**를 부여한다.

3) 스타일 구현 원칙
- 디자인 시안의 간격/높이/테두리/타이포그래피를 반영.
- 상태(state)별 스타일을 명확히 분리.
- 핵심 수치(`small` 높이 등)는 시안 기준값으로 고정.
- `styles.module.css`는 **레이아웃 / variant / state / 아이콘** 등 의미 단위로 **섹션 주석**을 두어 구역을 나눈다.
- 실제 DOM 상호작용이 가능한 컴포넌트는 `:hover`, `:focus`, `:focus-within`, `:active`, `:disabled`, `:has(...)` 등 **가상(논리) 선택자**로 상태를 표현하는 것을 우선한다.
- 에러 등은 `aria-invalid`와 조합해 스타일을 맞춘다.
- Storybook·정적 프리뷰에서만 “고정된 state”를 보여야 하면, 본문 CSS와 충돌하지 않게 **`[data-state]`(또는 `data-variant`) 폴백**을 둘 수 있다(실서비스 경로는 가상 클래스·ARIA 우선).
- 반복되는 선택자 나열은 `:is()` 등으로 묶어 **가독성**을 유지한다.
- variant×state 조합을 TSX에서 테마 클래스 맵으로만 쌓기보다, **CSS 변수(`--*`) + 모듈 클래스**로 정리할 수 있으면 그렇게 한다.

4) TSX와 클래스 조합(가독성·일관성)
- 최상위 래퍼의 `className`은 **한 변수에 모아서** 조합한다(예: `rootClass`, `wrapperClass`). CSS 모듈의 `styles.base`·`styles[variant]`·조건부 클래스·외부에서 받은 `className` prop을 **한곳에서만** 합친다.
- 합치는 방식은 프로젝트 표준에 따른다: **템플릿 리터럴**이거나, 레포에서 쓰는 `clsx` / `classnames` / `cn()` 등 **하나의 헬퍼**로 통일한다. JSX에 클래스 문자열을 여러 줄로 흩뿌리지 않는다.
- **다른 컴포넌트와 이름·패턴을 맞출 필요는 없다.** 이 컴포넌트 안에서만 위 규칙을 지키면 된다.
- 루트에 피그마/스토리 검증용으로 `data-variant`, `data-state` 등이 필요하면 부여한다(프로젝트 합의 범위 내).

5) 색상/토큰 원칙
- variant 의미를 시안 기준으로 일관되게 정의.
- 색상/타이포그래피는 전역 토큰(`globals.css` 또는 constants) 우선 사용.
- 임의 hex 하드코딩은 지양하고, 불가피하면 이유를 기록.

6) 아이콘 렌더 규칙(해당 시)
- 팀에서 쓰는 **아이콘 라이브러리(패키지)를 우선**하고, Figma에서 내보낸 정적 SVG는 스펙·프롬프트에 명시된 경우에만 사용한다.
- `showLeftIcon && leftIcon`일 때만 왼쪽 아이콘 노출.
- `showRightIcon && rightIcon`일 때만 오른쪽 아이콘 노출.
- 아이콘 슬롯 크기/정렬 기준(예: 20x20)을 명시. 공통 래퍼가 있으면 크기·선 두께를 그 기준으로 통일한다.

7) Storybook 연계 고려사항(해당 시)
- 주요 축(variant/size/theme/state) 검증이 가능하도록 구현.
- 지원하지 않는 조합은 임의 생성하지 않고 제외.
