사용법)
- 이 파일을 복사해서 경로/컴포넌트 기준만 교체해 사용.
- 피그마/컴포넌트에 없는 축은 스토리에서 제외하고, 필요한 스토리만 선별 구성.

요청)
- 아래 조건을 모두 적용해 컴포넌트 Storybook 스토리를 구현.
- 구현 결과를 체크리스트로 반환.

==============================================

조건-커서룰)
- 아래 커서룰을 적용하고, 작업 종료 시 적용 결과를 체크리스트로 반환.
  - @01-common.mdc

==============================================

조건-파일경로)
- 참고 TSX: [예: src/commons/components/{component-name}/index.tsx]
- 참고 CSS: [예: src/commons/components/{component-name}/styles.module.css]
- 구현 STORIES: [예: src/commons/components/{component-name}/index.stories.tsx]

==============================================

핵심요구사항)
1) CSF 기본 구조
- `title`, `component`, `tags`, `args`, `argTypes`, `parameters.layout` 포함.
- 기본 확인용 `Playground` 스토리 포함.

2) 컨트롤 구성 원칙
- 핵심 props를 컨트롤에 연결:
  - 필수 축: `variant`, `size`, `theme`, `state`
  - 선택 축: `type`, `intent`, `shape`, `fullWidth`, `nativeType` (해당 시)
  - 공통 옵션: `disabled`
  - 아이콘형 컴포넌트(해당 시): `showLeftIcon`, `showRightIcon`
- `ReactNode` props(`leftIcon`, `rightIcon`)는 샘플 기본값 제공 + `control: false`.
- 지원하지 않는 조합은 임의 생성하지 않고 제외.

3) 스토리 세트 구성
- 필수 스토리:
  - `Playground`
  - `Variants` (primary/secondary/tertiary 비교)
  - `Sizes` (small/medium/large 비교)
  - `Themes` (light/dark 비교)
  - `States` (default/hover/active/disabled 비교)
- 선택 스토리:
  - `Types` 등은 해당 축이 실제 존재할 때만 추가.
  - 필요 시 `FullWidth` 등 실사용 케이스 스토리 추가.

4) 검증 가능성
- 스토리만으로 팀원이 주요 UI/동작을 빠르게 검증할 수 있어야 함.
- 프로젝트의 variant 의미(예: primary/secondary/tertiary)를 스토리에서 명확히 확인 가능해야 함.

체크리스트(출력 포맷)
- [ ] 지정 경로에 스토리가 구현됨
- [ ] 필수 축(variant/size/theme/state) 스토리가 모두 포함됨
- [ ] 선택 축(type 등) 스토리가 필요한 범위로만 구성됨
- [ ] 핵심 props가 컨트롤에서 조작 가능함
- [ ] (해당 시) 아이콘 불린 토글이 컨트롤에서 동작함
- [ ] 팀원이 스토리만 보고 컴포넌트 상태를 검증할 수 있음
