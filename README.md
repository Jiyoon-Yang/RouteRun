# 🏃‍♀️ RunningCourse

사용자가 직접 러닝 코스를 등록하고 공유하는 **위치 기반 커뮤니티**입니다.  
**실제 서비스의 데이터 흐름과 사용자 경험**을 설계하는 데 집중한 프로젝트입니다.

---

## 🌿 협업 가이드 (필독!)

프로젝트의 효율적인 관리를 위해 **실무형 브랜치 전략**을 사용합니다.

### 📍 브랜치 역할

- **`main`**: 최종 배포용 브랜치 (직접 수정 절대 금지)
- **`dev` (Default)**: 모든 기능이 모이는 **중간 조립실**
- **`feat/기능명`**: 각자 기능을 개발하는 개인 작업 공간

### 🔄 작업 순서

1. **시작**: `dev`에서 최신 코드를 받고 내 브랜치를 생성합니다.
   - `git checkout dev && git pull origin dev`
   - `git checkout -b feat/기능명`
2. **개발**: 기능을 구현하고 내 컴퓨터에 저장(Commit)합니다.
3. **제출**: 깃허브에 올린 후 **`dev` 브랜치로 PR**을 날립니다.
4. **검토**: 팀원의 **승인(Approve) 1개 이상**이 있어야 머지가 가능합니다.
   - 본인이 올린 PR은 직접 합칠 수 없으며, 반드시 팀원이 확인해야 합니다.
5. **합치기**: 승인이 완료되면 `dev`에 최종적으로 합칩니다.

## 🧭 주요 기능 및 서비스 흐름

### 📱 페이지 구성 (IA)

- **홈 (/)**: 현재 위치 기반 지도 탐색 및 **거리별 필터**(3/5/10km)
- **코스 상세**: 경로 표시(**Polyline**), 상세 정보 확인 및 좋아요 기능
- **코스 등록**: 지도 클릭 기반 **실시간 경로 생성**, 좌표 2개 이상 시 등록 가능
- **마이페이지**: 프로필 관리 및 **내가 작성한 코스 목록** 수정/삭제

### 🧊 사용자 흐름 (User Flow)

- **탐색**: 지도 마커 클릭 → 바텀시트 정보 확인 → 상세 페이지 이동
- **생성**: 로그인 → 지도 클릭으로 경로 생성 → 정보 입력 → 등록 완료

## 🛠 기술 스택 및 데이터 설계

- **Frontend**: **Next.js**, Tailwind CSS / **Deploy**: Vercel
- **Map API**: Kakao Maps 또는 Google Maps (검토 중)
- **Database**: **현재 미정** (추후 결정 예정)

## 🚀 개발 단계 (Phase)

1. **Phase 1 (MVP)**: 인증 시스템, 코스 등록, 지도 목록/상세 페이지 구현
2. **Phase 2**: 좋아요 기능, 개인 보관함, 내 코스 관리 기능 고도화
3. **Phase 3**: 거리 자동 계산 로직 및 UI/UX 정교화
