조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TS 파일경로: host/src/pages/api/auth/anonymous-login.ts

==============================================

핵심요구사항) 익명 로그인(Anonymous Login) API를 구현할 것.
            1. 엔드포인트 구조
            - 메서드: POST 단일 엔드포인트
            - 역할: Supabase 익명 사용자 생성 및 세션 발급 처리
            2. 요청 데이터 (선택)
            - returnTo: 로그인 성공 후 이동할 경로 (클라이언트에서 전달 시)
            3. 인증 처리
            - 메서드: Supabase의 signInAnonymously() 메서드 사용
            - 성공조건: 응답에서 유효한 access token 및 refresh token 확보
            - 실패조건: 토큰 누락 또는 인증 생성 실패 시 에러 처리
            4. 브릿지 응답
            - 토큰전달: 클라이언트 상태 및 저장소 업데이트를 위해 발급된 토큰 데이터(access_token, refresh_token)를 JSON 형태로 응답 반환
            - (또는) 리다이렉트: 프론트엔드 처리 방식에 따라, 필요시 토큰 전달 후 returnTo 경로로 최종 이동할 수 있도록 지원
            5. 에러 처리
            - 인증 실패 시 명확한 에러 메시지와 함께 적절한 HTTP 상태 코드(예: 400 또는 500) 반환

추가-요구사항)
        - returnTo 정책: 사용 시 상대 경로만 허용
        - 인증옵션: 서버 클라이언트는 세션 자동 저장/자동 갱신 비활성화
        - 클라이언트 생성: src/lib/supabase/initialize/index.ts 유틸 import 사용 (또는 프로젝트 환경에 맞는 초기화 경로 사용)
        - 금지 사항: 각 파일에서 createClient 직접 생성 금지
