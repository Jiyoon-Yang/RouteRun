조건-파일경로) 구현될 TS 파일경로: src/app/api/auth/logout/route.ts

==============================================

핵심요구사항) 로그아웃 API를 구현할 것.
            1. 메서드
                - 허용: POST
                - 그 외: 메서드 오류 처리
            2. 인증 기준
                - 입력헤더: Authorization Bearer access token
                - 오류처리: 헤더 누락/형식 오류는 잘못된 요청
            3. 서버 처리
                - 방식: Supabase 로그아웃은 가능하면 처리하고, 실패해도 전체 로그아웃 흐름은 유지
            4. 응답 정책
                - 원칙: 일부 내부 오류가 있어도 클라이언트 관점 로그아웃 성공(멱등성)
            5. 제외 범위
                - 리프레시 재발급 로직 미구현

추가-요구사항) Supabase 클라이언트 사용 규칙
            - 인증옵션: 세션 자동 저장/자동 갱신 비활성화 유지
            - import 기준: src/lib/supabase/initialize/index.ts 유틸 import 사용
            - 금지 사항: 각 파일에서 createClient 직접 생성
