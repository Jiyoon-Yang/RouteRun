# 개요 및 만들어진 과정

## 이 기능이 하는 일

- **목적**: 러닝 코스 등록 시, 사용자가 지도에서 **출발지 → 경유지(여러 개) → 도착지**를 클릭해서 지정할 수 있도록 하는 **테스트용** 화면입니다.
- **역할**: 실제로 Supabase에 저장하기 전에, 카카오맵 API 연동·좌표 수집·데이터 구조를 확인하고 싶을 때 사용합니다.
- **결과**: 클릭한 좌표가 순서대로 배열에 쌓이고, 각 점이 START / WAYPOINT / END 로 구분되며, Supabase 테이블에 넣을 수 있는 형태로 변환됩니다.

---

## 만들어진 과정 (단계별)

### 1단계: 기본 환경 준비

- **라우트**: `/temp` 에 접속하면 테스트 페이지가 보이도록 `src/app/temp/page.tsx` 생성.
- **타입**: 카카오맵은 CDN 스크립트로 로드되므로, `window.kakao` 를 TypeScript에서 쓰기 위해 `src/types/kakao.d.ts` 에 전역 타입 선언 추가 (Map, LatLng, Polyline, Marker, Event 등).

### 2단계: 지도 띄우기 + 클릭으로 좌표 수집

- **지도 표시**: Next.js `Script` 로 카카오맵 SDK 로드 (`autoload=false`), 로드 완료 후 `kakao.maps.load()` → `kakao.maps.Map()` 으로 지도 생성. 초기 중심은 서울역 근처.
- **클릭 이벤트**: 지도에 `click` 리스너를 달아, 클릭할 때마다 `mouseEvent.latLng` 에서 위도·경도를 꺼내 콘솔에 출력하고, `points` 상태 배열에 순서대로 추가.

### 3단계: 좌표 타입 구분 및 Supabase 형식 변환

- **타입 구분**: 배열에 들어간 순서만으로 구분.  
  **첫 번째 = START**, **마지막 = END**, **그 사이 = WAYPOINT**.
- **변환 함수**: `toSupabaseCoursePoints(courseId, points)` 로 `points` 를 Supabase 코스 포인트 테이블 행 배열로 변환 (id 제외, course_id, point_type, latitude, longitude, sequence 포함).  
  이 변환 결과를 콘솔에 출력해 API 응답 구조를 파악할 수 있게 함.

### 4단계: 지도 위 시각화 (선 + 마커)

- **폴리라인**: `points` 가 2개 이상일 때, 순서대로 잇는 **검은색 선**을 카카오맵 `Polyline` API 로 그림. `points` 가 바뀔 때마다 기존 선을 지우고 다시 그림.
- **마커**: 각 좌표마다 `Marker` 를 찍어서 출발지·경유지·도착지 위치를 지도 위에 표시.

### 5단계: 출발지 = 도착지 (같은 위치로 도착지 지정)

- **요구사항**: 도착지를 출발지와 같은 곳으로 두고 싶을 수 있음.
- **구현**:  
  - **출발지(첫 번째) 마커**에 마우스를 올리면 `setOpacity(1)` 로 진하게 표시.  
  - 출발지 마커를 **클릭**하면, 출발지와 **같은 좌표**를 한 번 더 배열에 넣어서 마지막 점이 도착지(END)가 되도록 함.  
  - 지도 빈 곳을 클릭했을 때, 그 좌표가 출발지와 매우 가까우면(오차 0.0001) 중복 추가를 막고, 출발지 마커 클릭에서만 “같은 위치 = 도착지”가 추가되도록 처리.

### 6단계: 환경 변수 및 키 정리

- **키**: 카카오맵 **JavaScript 키**를 사용해야 함 (REST API 키 사용 시 "wrong appKey format" 발생).
- **노출**: 브라우저에서 스크립트에 키를 넣어야 하므로 `.env` 의 `NEXT_PUBLIC_KAKAO_MAP_API_KEY` 로 넣고, `MapTest` 에서 해당 환경 변수를 사용하도록 수정.

## 데이터 흐름 요약

```
사용자 지도/마커 클릭
    → 클릭 좌표를 points 배열에 추가 (또는 출발지 마커 클릭 시 같은 좌표를 END로 추가)
    → points 변경 시: 폴리라인·마커 다시 그림 + toSupabaseCoursePoints() 로 변환
    → 변환 결과는 화면 목록 + 콘솔에서 확인 (실제 DB 저장은 이 테스트 범위 밖)
```
