# 기능: 지도 표시와 클릭 좌표 수집

지도를 화면에 띄우고, 사용자가 클릭한 위치의 위도·경도를 수집하는 흐름을 설명합니다.

---

## 1. 지도가 뜨기까지의 순서

1. **스크립트 로드**  
   `MapTest` 에서 `next/script` 의 `Script` 로 카카오맵 SDK URL을 넣습니다.  
   - URL에 `autoload=false` 를 붙여, 스크립트만 로드하고 지도는 자동으로 생성되지 않게 합니다.  
   - `strategy="afterInteractive"` 로 페이지가 대화 가능해진 뒤 로드합니다.

2. **로드 완료 감지**  
   `Script` 의 `onLoad` 콜백에서 `setScriptLoaded(true)` 를 호출합니다.

3. **지도 생성**  
   `scriptLoaded` 가 true 인 `useEffect` 안에서:  
   - `kakao.maps.load(() => { ... })` 를 호출합니다.  
   - 콜백 안에서 `mapContainerRef.current`(div)에 `new kakao.maps.Map(container, options)` 로 지도를 만듭니다.  
   - 초기 `options`: 중심은 서울역 근처(37.5563, 126.9723), `level: 5`.

4. **지도 인스턴스 보관**  
   생성한 지도 객체를 `mapRef.current` 에 넣고, `setMapReady(true)` 로 “지도 준비 완료” 상태를 둡니다.  
   이후 다른 `useEffect` 에서 폴리라인·마커를 그릴 때 이 `mapRef` 를 사용합니다.

---

## 2. 클릭 시 좌표 수집

- **이벤트 등록**  
  지도 생성 직후 `kakao.maps.event.addListener(map, "click", handler)` 로 클릭 리스너를 단다.

- **핸들러 동작**  
  - `mouseEvent.latLng.getLat()`, `mouseEvent.latLng.getLng()` 로 위도·경도를 구한다.  
  - 이 좌표가 **출발지(첫 번째 점)와 거의 같은지** 확인한다.  
    - 같은 경우(오차 0.0001 이내): 나중에 “출발지 마커 클릭”에서 도착지로 넣기 위해, 여기서는 **추가하지 않고 return** 한다.  
    - 다른 경우: `console.log` 로 좌표를 출력하고, `setPoints((p) => [...p, { latitude, longitude }])` 로 `points` 배열에 한 개 추가한다.

- **최신 points 참조**  
  클릭 핸들러는 한 번 등록된 뒤로 바뀌지 않으므로, 안에서 `points` 를 직접 쓰면 예전 값(클로저)을 보게 될 수 있다.  
  그래서 `pointsRef.current = points` 로 항상 최신 배열을 가리키게 하고, 핸들러 안에서는 `pointsRef.current` 로 “출발지와 같은지” 여부를 판단한다.

---

## 3. 요약

| 단계 | 하는 일 |
|------|---------|
| Script 로드 | 카카오맵 SDK 스크립트 삽입, `autoload=false` |
| onLoad | `scriptLoaded = true` |
| useEffect (scriptLoaded) | `kakao.maps.load()` → 지도 생성 → `mapRef`·`mapReady` 설정 → 지도 click 리스너 등록 |
| 지도 클릭 | 위도·경도 조회 → (출발지와 같지 않으면) 콘솔 출력 + `points` 에 추가 |

이렇게 쌓인 `points` 가 [04-기능-경로선-마커-출발지도착지.md](./04-기능-경로선-마커-출발지도착지.md) 의 선·마커와 [05-기능-Supabase-변환.md](./05-기능-Supabase-변환.md) 의 변환 로직에 사용됩니다.
