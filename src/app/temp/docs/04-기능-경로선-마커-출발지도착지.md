# 기능: 경로 선(폴리라인), 마커, 출발지=도착지

클릭으로 쌓인 `points` 를 지도 위에 **선**과 **마커**로 그리며, **출발지와 같은 위치를 도착지**로 두는 방법을 설명합니다.

---

## 1. 언제 그리는가

- **트리거**: `points` 또는 `mapReady` 가 바뀌는 `useEffect` 에서 실행됩니다.
- **조건**: `mapReady` 가 true 이고 `mapRef.current` 와 `window.kakao` 가 있을 때만 동작합니다.

---

## 2. 기존 요소 제거

- **폴리라인**: `polylineRef.current` 가 있으면 `setMap(null)` 로 지도에서 제거한 뒤 ref 를 null 로 둡니다.
- **마커**: `markersRef.current` 에 있는 모든 마커에 `setMap(null)` 을 호출하고, 배열을 비웁니다.  
  그다음에 새로 그리기 때문에, 좌표를 수정·삭제하지 않고 “전부 다시 그리기” 방식으로 동작합니다.

---

## 3. 경로 선(폴리라인)

- **그리는 조건**: `points.length >= 2` 일 때만 선을 그립니다.
- **구성**: `points` 를 순서대로 `kakao.maps.LatLng` 배열로 바꾼 뒤, `new kakao.maps.Polyline({ path, strokeWeight: 4, strokeColor: "#000000", strokeOpacity: 0.9, strokeStyle: "solid" })` 로 생성합니다.
- **표시**: `polyline.setMap(map)` 으로 지도에 붙이고, 인스턴스를 `polylineRef.current` 에 저장해 다음 갱신 시 제거할 수 있게 합니다.

---

## 4. 마커

- **각 점마다**: `points.forEach` 로 각 좌표에 `new kakao.maps.Marker({ position, map })` 를 만들어 지도에 표시합니다.
- **저장**: 만든 마커를 `markersRef.current` 에 push 해 두어, 다음에 `points` 가 바뀌었을 때 전부 `setMap(null)` 로 지울 수 있게 합니다.

---

## 5. 출발지 마커만의 동작 (호버·클릭)

- **대상**: `index === 0` 인, 즉 **첫 번째(출발지)** 마커만 추가 이벤트를 단다.

- **호버 시 “진하게”**  
  - 기본: `marker.setOpacity(0.85)`  
  - `mouseover`: `marker.setOpacity(1)`  
  - `mouseout`: `marker.setOpacity(0.85)`  
  → 출발지 마커에 마우스를 올리면 더 진하게 보이도록 했습니다.

- **클릭 시 “같은 위치를 도착지로 추가”**  
  - `click` 리스너에서 `points[0]`(출발지 좌표)를 그대로 한 번 더 `setPoints` 로 배열 끝에 넣습니다.  
  - 그러면 마지막 점이 출발지와 동일한 좌표가 되고, `toSupabaseCoursePoints` 에서 그 점이 END 로 매핑됩니다.  
  - 지도 빈 곳을 클릭했을 때는, 클릭 좌표가 출발지와 매우 가까우면(0.0001 오차) 아예 추가하지 않아서, “출발지 위치 클릭 = 출발지 마커 클릭”으로만 도착지가 추가되도록 했습니다.

---

## 6. 요약

| 항목 | 내용 |
|------|------|
| 선 | `points.length >= 2` 일 때만, 순서대로 이어지는 검은 폴리라인 |
| 마커 | 모든 점에 기본 마커, 출발지(첫 번째)만 opacity·이벤트 추가 |
| 출발지=도착지 | 출발지 마커 호버 → 진해짐, 클릭 → 같은 좌표가 배열 끝(END)으로 추가 |

이렇게 갱신된 `points` 는 [05-기능-Supabase-변환.md](./05-기능-Supabase-변환.md) 에서 Supabase 테이블 형식으로 변환됩니다.
