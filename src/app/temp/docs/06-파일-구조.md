## 파일별 역할

### `src/app/temp/page.tsx`

- **역할**: `/temp` URL에 접속했을 때 보이는 페이지 컴포넌트.
- **내용**: 상단에 `MapTest` 한 번만 렌더하고, 레이아웃용 `main` 으로 감쌈. 배경·글자 색만 블랙·화이트로 지정.

---

### `src/app/temp/MapTest.tsx`

- **역할**: 코스 등록 테스트의 실제 UI·로직. **Client Component** (`"use client"`).
- **주요 내용**:
  - 카카오맵 SDK 스크립트 로드 (`next/script`), 로드 후 지도 생성·클릭 리스너 등록.
  - `points` 상태로 클릭 좌표 배열 관리.
  - `points`·`mapReady` 변경 시 폴리라인·마커 다시 그리기, 출발지 마커 호버·클릭 처리.
  - `toSupabaseCoursePoints(courseId, points)` 로 Supabase 행 배열 생성.
  - 화면: 지도 영역 + “클릭한 좌표” 목록(sequence, point_type, 위도·경도).
- **의존**: `src/types/kakao.d.ts`(전역 타입), 환경 변수 `NEXT_PUBLIC_KAKAO_MAP_API_KEY`.

---

### `src/types/kakao.d.ts`

- **역할**: 카카오맵 JavaScript SDK가 `window.kakao` 로 주입되므로, TypeScript에서 해당 객체를 타입으로 쓰기 위한 **전역 선언**.
- **포함 내용**: `KakaoMapsLatLng`, `KakaoMapsMap`, `KakaoMapsPolyline`, `KakaoMapsMarker`, `KakaoMapsEvent`, `KakaoMapsStatic` 등 사용하는 API만 최소한으로 선언. `declare global { interface Window { kakao: ... } }` 로 전역 보강.

---

### `src/app/temp/docs/` 아래의 md 파일들

- **README.md**: 문서 진입점. 목차, 한 줄 요약, 사용 방법, 필요 설정 요약.
- **01-개요-및-만들어진-과정.md**: 기능이 무엇인지, 어떤 단계로 만들어졌는지.
- **02-사용-기술.md**: Next.js, React, 카카오맵, Tailwind, 환경 변수, 타입 등 사용 기술 정리.
- **03~05**: 기능별 상세 설명 (지도·클릭 / 경로선·마커·출발지=도착지 / Supabase 변환).
- **06-파일-구조.md**: 이 파일. 프로젝트 내 관련 파일 위치와 역할.

## 설정 파일·환경

- **`.env`** (프로젝트 루트, 보통 .gitignore 대상):  
  `NEXT_PUBLIC_KAKAO_MAP_API_KEY` 에 카카오맵 **JavaScript 키** 설정.  
  문서화만 하고 저장소에는 넣지 않는 것이 일반적입니다.

이 구조를 기준으로 [01~05번 문서](./01-개요-및-만들어진-과정.md)의 “어디서 무엇이 일어나는지”를 대응해서 읽으면 됩니다.
