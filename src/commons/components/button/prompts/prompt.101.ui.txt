핵심요구사항)
1) 컴포넌트 API
- 아래 props를 지원할 것
  - `variant: 'primary' | 'secondary' | 'tertiary'`
  - `type: 'fill' | 'outline'`
  - `state: 'default' | 'hover' | 'active' | 'disabled'`
  - `size: 'small' | 'medium' | 'large' | 'x-large'`
  - `fullWidth?: boolean`
  - `nativeType?: 'button' | 'submit' | 'reset'`
  - `disabled?: boolean`
  - `leftIcon?: ReactNode`
  - `rightIcon?: ReactNode`
  - `showLeftIcon?: boolean` (기본 false)
  - `showRightIcon?: boolean` (기본 false)

2) 아이콘 표시 규칙
- `showLeftIcon && leftIcon`일 때만 왼쪽 아이콘을 렌더링할 것
- `showRightIcon && rightIcon`일 때만 오른쪽 아이콘을 렌더링할 것
- 아이콘 래퍼는 20x20 기준으로 정렬할 것

3) 사이즈 기준
- `small`의 높이는 피그마 기준으로 `36px`를 사용할 것
- 나머지 size도 기존 토큰 기반 규칙을 유지할 것

4) variant/색상 기준 (현재 프로젝트 기준 고정)
- `primary`는 black 세트(black node 기준)로 매핑할 것
- `secondary`는 blue 세트(blue node 기준)로 매핑할 것
- `tertiary`는 red 세트(red node 기준)로 매핑할 것
- `fill`/`outline` 모두 state(default/hover/active/disabled) 별 색이 피그마 세트와 일치해야 함

5) 토큰 사용 규칙
- 색상/타이포그래피는 `src/app/globals.css`의 CSS 변수 토큰을 우선 사용
- 가능한 한 하드코딩 hex를 사용하지 말 것
