아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-구현목표) Input 하단 부가 문구(`addtional_text`)를 독립 공통 컴포넌트로 구현할 것.
            - 피그마 공통컴포넌트 기준으로 상태별 스타일을 반영할 것
            - 구현 결과(완료/미완료)를 체크리스트로 반환할 것

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이후 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc
            - @03-ui.mdc

==============================================

조건-피그마) 활용할MCP: Figma Desktop MCP

==============================================

핵심요구사항) 부가 문구 텍스트는 상태별 스타일을 지원할 것.
            - 상태는 `default | success | error`를 지원할 것
            - 아이콘 노출은 `showIcon` boolean props로 제어할 수 있게 구현할 것
            - `message` props로 문구를 전달받고, 빈 문자열이면 렌더링을 생략할 수 있게 구현할 것
            - Input 조합 컴포넌트에서 재사용 가능한 단일 책임 구조로 구현할 것

핵심요구사항) 연동된 MCP 링크에 맞는 피그마의 디자인을 정확히 소스코드로 구현할 것.
