핵심요구사항)
1) 축 설계
- 타입 축: `type` = 'none' | 'optional' | 'required' | 'info'
- 툴팁 축: `tooltip`(string), `tooltipTrigger` = 'hover' | 'click' | 'both'
- 사이즈 축은 두지 않고 단일 타이포 스케일로 고정.

2) 컴포넌트 API 원칙
- `label` 요소 기반으로 구현.
- `htmlFor`를 props로 받아 input과 연결 가능하게 구현.
- `type` 값에 따라 보조 표기/의미를 반영:
  - `none`: 보조 표기 없음
  - `optional`: 선택 표기
  - `required`: `*` 표기
  - `info`: 정보성 타입 표현
- 아이콘은 `type='info'`일 때 자동 노출.
- 툴팁은 `type='info' && tooltip`일 때만 노출.
- `tooltipTrigger` 값에 따라 트리거 동작을 분기:
  - `hover`: 마우스 오버/포커스 시 노출
  - `click`: 클릭 토글로 노출
  - `both`: hover + click 모두 지원
- 타입은 유니온 타입 우선, `any` 사용 금지.

3) 스타일 구현 원칙
- 피그마 기준 타이포/간격/색상/타입 표현을 반영.
- 타입(type)별 스타일을 명확히 분리.
- Input 조합 컴포넌트에서 재사용 가능한 단일 책임 구조로 유지.

4) 토큰 원칙
- 색상/타이포그래피는 전역 토큰 우선 사용.
- 임의 hex 하드코딩은 지양하고, 불가피하면 이유를 남김.
