핵심요구사항) 컴포넌트는 실제 `<label>` 요소 기반으로 구현할 것.
            - 프리뷰/스토리북 전용 강제 상태 구조와 JS props로 시각 상태를 고정하지 말 것

핵심요구사항) props는 표준 호환 + 기존 API를 유지할 것.
            - htmlFor로 input과 연결 가능하게 할 것
            - type은 'none' | 'optional' | 'required' | 'info' 유니온 타입을 유지할 것
            - tooltip(string), tooltipTrigger('hover' | 'click' | 'both') props 시그니처는 유지하되 기능 구현에는 사용하지 않을 것
            - any 타입 사용 금지

핵심요구사항) Type
            - none: 보조 표기 없음
            - optional: '(선택)' 표기
            - required: '*' 표기
            - info: 정보성 타입 + 아이콘만 노출 (툴팁 기능 제외)
            - required/optional 표시는 [data-type] 또는 의미 클래스 선택자로 매핑할 것

핵심요구사항) tooltip 기능(hover/click/both 토글, 열림 상태 제어)은 구현 범위에서 제외할 것.
            - info 타입에서는 아이콘만 렌더링하고 툴팁 기능은 만들지 말 것

핵심요구사항) 접근성(Accessibility)과 표준 속성을 함께 만족할 것.
            - htmlFor/id 연결로 레이블링을 보장할 것
            - required 문맥에서는 의미 전달이 가능하도록 텍스트/속성을 함께 고려할 것
            - info 아이콘이 인터랙션이 없다면 버튼 역할을 부여하지 말 것

핵심요구사항) 스타일 범위는 Figma 단일 label 필드로 제한할 것.
            - size/state variant prop을 추가하지 말 것
            - 기본 클래스 + 가상클래스/속성 선택자 중심으로 구성할 것
