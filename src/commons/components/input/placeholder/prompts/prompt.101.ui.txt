핵심요구사항)
- API: `placeholder`, `value?`, `state`, `variant`, `showLeftIcon?`, `showRightIcon?`, `leftIcon?`, `rightIcon?`, `disabled?`, `readOnly?`, `onChange?`.
- `showLeftIcon` / `showRightIcon` 기본값은 true.
- `state`: `default | hover | focus | focus_none | filled | error | success | disabled` — **focus_none**은 빈 값+포커스(placeholder 숨김), **focus**는 시안상 placeholder 유지, **success**는 그린 보더.
- `variant`: `primary | secondary`.
- `disabled === true`이면 항상 `disabled` 상태로 처리한다.
- `state === 'filled'` 또는 `success`일 때 input value에 `value`가 비어 있으면 `placeholder` 문자열을 값처럼 넣는다.
- 그 외 상태에서는 값이 있을 때만 표시·그 외 빈 문자열+placeholder.
- 루트 `className`은 `styles.base` + `styles.primary`|`styles.secondary` + (선택) 상태 모듈을 조합한다. `STATE_CLASS`에 `stateHover`·`stateFocus`·`stateFilled`·`stateError`·`stateSuccess`·`stateDisabled`(`default`는 추가 없음).
- 루트에 `data-variant`, `data-state`는(선택) 디버그·스토리용. `state === 'error'`일 때는 `aria-invalid`를 루트에 부여한다.
- input은 `readOnly`(프리뷰·스토리북용).
- 색·타이포는 `src/app/globals.css` 토큰(`var(--color-*)`, `--typography-body-1-ko-desktop-*`)으로 맞춘다.
- 필드 높이·패딩·라운드는 단일 스케일이다(size variant 없음).
- `styles.module.css`: variant 섹션 **default → filled → :hover → :focus-within(focus 보더) → success → error → disabled** 순. `:hover:not(.stateError):not(.stateDisabled):not(.stateSuccess)` 등.
- 아이콘은 팀에서 사용하는 아이콘 라이브러리를 우선 사용한다. 필드 슬롯에는 `FieldLucideIcon`(`src/commons/icons/`)으로 크기·선 두께를 통일할 것.
- `disabled`일 때 input `disabled` + 포인터 차단 등 비활성 표현이 분명하다.
