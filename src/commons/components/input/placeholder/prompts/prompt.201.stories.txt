핵심요구사항)
1) 기본 설정
- Storybook CSF 형식으로 작성할 것
- `title`, `component`, `tags`, `args`, `argTypes`, `parameters.layout`를 포함할 것

2) 컨트롤 항목
- 아래 항목은 컨트롤 가능해야 함
  - `placeholder` (text)
  - `value` (text)
  - `state` (`default | hover | focus | filled | error | disabled`)
  - `variant` (`primary | secondary`)
  - `disabled` (boolean)
  - `showLeftIcon` (boolean)
  - `showRightIcon` (boolean)
- `showLeftIcon`, `showRightIcon`은 값에 따라 실제 `leftIcon`, `rightIcon` Props에 위 Figma export SVG(`import`한 URL을 `<img src={...}>`)가 전달되도록 `render`를 구현할 것
- `leftIcon`, `rightIcon`은 controls 테이블에서 숨길 것(`table.disable`)

3) 필수 스토리 구성
- `Playground`
- `FullWidth` (컨테이너 너비를 넓혀 필드 `width: 100%` 동작 확인)
- `Variants` (`primary` / `secondary` 한 화면 비교, 동일 `state`)
- `States` (`state` 전 구간을 한 화면에서 비교. 컴포넌트는 size variant 없음)
- `Filled` (`state === 'filled'`일 때 `value ?? placeholder`가 input에 보이는지 확인)
- `Disabled` (`disabled` prop 또는 `state === 'disabled'`로 비활성 표현 확인)

4) 의도 고정
- 스토리 설명(description) 또는 주석으로 아래를 명시할 것
  - `state`는 CSS 클래스 기반 시각 프리뷰용(실제 `:hover`가 아닌 prop으로 고정)
  - `disabled === true`이면 `state`와 무관하게 비활성 처리됨
  - `filled`일 때만 입력값 영역에 문자열이 채워짐
- 스토리만 봐도 variant / state / 아이콘 토글 / filled·disabled를 검증할 수 있어야 함
