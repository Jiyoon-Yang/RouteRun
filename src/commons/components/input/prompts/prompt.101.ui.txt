핵심요구사항)
1) 컴포넌트 API
- 아래 props를 유지/지원할 것
  - `label?: string`
  - `required?: boolean`
  - `state?: 'default' | 'hover' | 'filled' | 'active' | 'error' | 'disabled'`
  - `addtionalText?: string`
  - `addtionalTextState?: 'default' | 'success' | 'error' | 'disabled'`
  - `disabled?: boolean`
  - `showLeftIcon?: boolean` (기본 true)
  - `showRightIcon?: boolean` (기본 true)

2) 조합형 구조 유지
- Input은 래퍼 컴포넌트로 구현하고 하위 3종을 조합할 것
  - `label`
  - `placeholder`
  - `addtional_text`

3) 스타일 구현
- 피그마 기준 간격/높이/라운드/타이포를 반영할 것
- 입력 필드 좌/우 아이콘은 `showLeftIcon` / `showRightIcon` 불린으로 노출 여부를 제어할 것
- 상태별 보더/텍스트 색상 차이를 분리할 것

4) 상태 및 문구 규칙
- `default | hover | filled | active | error | disabled` 상태가 모두 동작해야 함
- `addtional_text`는 상태별 색상을 반영해야 함
- `error` 상태에서는 에러 톤(보더/보조문구)이 명확해야 함

5) 접근성 및 토큰 사용
- `label`과 input을 `htmlFor`/`id`로 연결할 것
- 색상/타이포는 전역 토큰(`globals.css`) 기반으로 적용할 것
