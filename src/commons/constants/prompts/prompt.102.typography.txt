핵심요구사항)
- 피그마 foundation 내 typography 값을 “토큰화”하여 코드에서 단일 소스 오브 트루스로 관리할 것.
- foundation 내 typography 토큰 이름을 누락 없이 기재/반영할 것.
- foundation에 존재하는 모바일/데스크톱 분기 값을 동일 토큰 이름 체계를 유지한 채 TS와 CSS에 반영할 것.
- (중요) 추후 영문 typography는 한국어 값과 분리되어 교체/확장될 수 있도록(예: ko/en 분리 또는 확장 포인트 제공) 셋업할 것.
- CSS에서도 동일한 토큰 이름으로 접근 가능하도록 `globals.css`에 CSS 변수(`--typography-*`)를 셋업할 것.

typography 토큰 추출)
- typography 토큰명·목록은 프롬프트에 적지 않고, 입력-피그마)의 foundation 노드ID만으로 결정한다.
- CursorTalkToFigmaMCP에서 해당 노드에 정의된 typography 토큰 이름과 각 토큰의 속성(font-family, font-size, line-height, font-weight, letter-spacing 등) 값을 추출한 뒤, 그 결과를 `typography.ts`와 `globals.css`에 그대로 반영할 것.
- CSS 변수는 토큰 1개당 속성을 분해해 `--typography-${token}-font-family`, `--typography-${token}-font-size` ... 형태로 선언하는 것을 기본으로 한다. (프로젝트가 채택한 명명 규칙이 있으면 그 규칙을 우선)
