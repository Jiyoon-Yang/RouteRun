구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
- .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TS 파일경로: src/components/login/hooks/index.anonymous.login.hook.ts

==============================================

핵심요구사항) 익명 로그인 트리거 훅을 구현할 것.
            1. 트리거 역할
            - 대상: 익명 로그인 API 호출 (POST 요청)
            2. 성공 처리
            - 조건: API 응답 성공 + 토큰(access_token, refresh_token) 존재
            - 동작 1: 발급받은 토큰을 로컬 스토리지에 저장 (또는 Auth Provider의 로그인 처리 함수 호출)하여 즉시 인증 상태 업데이트
            - 동작 2: 응답받은 returnTo 경로(없을 경우 기본 홈 경로)로 라우팅 이동
            3. 파라미터 전달
            - 항목: returnTo (선택)
            - 목적: 로그인 완료 후 사용자를 원래 보던 페이지로 복귀시키기 위함
            4. 상태 노출
            - 항목: 요청 중 상태(isLoading) + 에러 상태(error)
            - 목적: 화면(UI)에서 로딩 스피너를 띄우거나 에러 메시지를 표시할 수 있도록 지원

추가-요구사항)
            - 호출 엔드포인트: /api/auth/anonymous-login (앞서 생성한 익명 로그인 API 경로 사용)
            - 금지 사항: 훅 내부에서 Supabase SDK(createClient 등) 직접 호출 절대 금지, 오직 API 라우트만 호출할 것
