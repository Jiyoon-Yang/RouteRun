아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - .cursor/rules/01-common.mdc

==============================================

조건-파일경로) 구현될 TS 파일경로: src/lib/supabase/auth/index.ts

==============================================

핵심요구사항) API 인가 유틸을 구현할 것.
            1. 인증 기준
                - 입력헤더: Authorization Bearer token
                - 오류처리: 헤더 누락/형식 오류는 인증 실패
            2. 유효성 검증
                - 방식: Supabase 사용자 조회 기반
                - 결과: 유효한 사용자 식별값 반환 가능
            3. 반환 형태
                - 기본형: 사용자 식별값 반환
                - 확장형: 사용자 전체 정보 반환
                - 토큰전용형: 토큰 유효성만 검증하고 사용자 식별값 반환
            4. 에러 처리
                - 방식: 공통 에러 타입
                - 분기: 코드 기반 처리 가능

추가-요구사항) 아래 에러 코드를 유지할 것.
            - UNAUTHENTICATED

추가-요구사항) Supabase 클라이언트 사용 규칙
            - import 기준: src/lib/supabase/initialize/index.ts 파일
            - 사용 기준: 토큰 검증이 필요한 함수는 토큰 기반 클라이언트 생성 유틸 우선 사용
            - 금지 사항: 각 파일에서 createClient 직접 생성
